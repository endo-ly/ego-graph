## 設計原則

### プライバシーファースト

すべての設計判断において、プライバシー保護を最優先する。

- 機密データはPrivate DB（NAS）に隔離
- LLM送信前に必ずマスク処理
- データは自己管理、第三者サービスへの依存を最小化

### ストレージ責務分離

SQLite と DuckDB / Parquet を責務で分離する。

- 更新系・会話履歴・運用メタデータ → SQLite
- 分析・集計・横断参照 → DuckDB
- 分析用データセット・長期保存 → Parquet

### 段階的実装

MVPから始め、段階的に機能拡張する。
